# Política de preservação e arquivamento

## Objeto preservado

A série `Modelo Axion Lotofácil: relatórios técnicos nº 1–4`, versão 1.0, é tratada como um objeto documental único. A edição contém quatro relatórios técnicos, um manifesto de integridade, metadados de citação e um pacote agregado.

## Imutabilidade

Após a publicação da Release e do registro Zenodo, os arquivos da versão 1.0 não devem ser substituídos silenciosamente. Alterações substantivas exigem uma nova versão numerada.

## Verificação

A integridade dos PDFs é verificada por `CHECKSUMS.sha256`. O pacote ZIP possui checksum próprio. Divergências devem ser tratadas como falha de integridade ou como indicação de versão diferente.

## Redundância

A preservação utiliza:

1. histórico Git e tag anotada;
2. GitHub Release com anexos;
3. Zenodo com DOI específico e DOI conceitual;
4. cópia local do pacote e dos checksums.

## Citação

- Citar o DOI específico quando a análise depender da versão 1.0.
- Citar o DOI conceitual para referência geral à série e suas atualizações.
